# 中文说明

## 这个 skill 解决什么问题

`gp-wiki-api` 把 GP Wiki 和 GP API 手册同步成本地资料库，让 Codex 在回答绿洲启元 / 和平精英 UGC 开发问题时，可以直接基于本地文档检索和引用，而不是临时浏览网页。

它主要用于：

- 查询 Lua API、类、函数、枚举、结构体字段
- 查询玩法系统、UI、联网、观战、社交、AI、商业化等文档
- 根据关键词定位原始文档和本地镜像文件
- 在用户要求“最新文档”时重新同步远端资料

## 推荐工作流

1. 先判断问题属于 Wiki 文章还是 API 手册。
2. 用 `rg` 搜索索引：
   - Wiki：`references/article-index.tsv`
   - API：`references/api-manual/symbol-index.tsv`
3. 只打开命中的 markdown 文件。
4. 回答时说明：
   - 引用的本地文件
   - 原始来源 URL
   - 哪些是文档事实，哪些是推断

## 关键文件

- `SKILL.md`：Codex 触发和使用该 skill 的核心说明
- `references/article-index.md`：Wiki 文章目录
- `references/article-index.tsv`：适合命令行检索的 Wiki 索引
- `references/api-manual/symbol-index.md`：API 符号目录
- `references/api-manual/symbol-index.tsv`：适合命令行检索的 API 索引
- `references/sync_usage.md`：同步脚本参数说明
- `scripts/sync_gp_docs.py`：同步全部资料的入口脚本

## 同步命令

刷新全部资料：

```powershell
python scripts/sync_gp_docs.py --force
```

跳过 Wiki，只刷新 API 手册：

```powershell
python scripts/sync_gp_docs.py --skip-wiki --force
```

跳过 API 手册，只刷新 Wiki：

```powershell
python scripts/sync_gp_docs.py --skip-api-manual --force
```

API 手册首轮同步文件较多，耗时可能较长。脚本支持断点续跑，失败后重新执行同一命令即可。

## 示例检索

```powershell
rg "APlayerController|UGCPlayerControllerSystem" references/api-manual/symbol-index.tsv references/api-manual/symbols
rg "网络同步|RPC|EventSystem" references/article-index.tsv references/articles
rg "背包系统|物品编辑器|通用掉落" references/article-index.tsv references/articles
```

## 维护注意事项

- 不要提交 `__pycache__`、`.pyc` 或本地临时文件。
- 如果远端文档结构变化，优先修复同步脚本，再重建索引。
- 大文件索引不要整篇塞进上下文，应该先搜索再按需读取。
- 如果文档中出现乱码，优先检查源站编码和同步脚本的 markdown 渲染逻辑。
