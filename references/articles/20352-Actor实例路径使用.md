# Actor实例路径使用

Actor 实例路径功能用于在UGC编辑器中快速引用场景内的指定实例。开发者可以通过复制实例路径，并在运行时基于该路径查找对应的 Actor ，从而高效完成对象引用与操作。

在世界大纲或场景中可被选中的对象，均属于 Actor 实例。

<br>

## 功能概述

Actor 实例路径提供了一种统一、简洁的方式，用于在编辑器与运行时之间建立实例引用关系。
相比传统手动维护引用的方式，该功能具有以下优势：

- 支持Lua配置直接引用：在配置中直接填写实例路径，无需额外绑定
- 便于关卡脚本定位对象：可快速查找预设点位、触发器、机关等
- 降低维护成本：场景调整时无需同步修改代码引用逻辑，减少手工维护引用关系的成本，提高开发效率
- 路径统一规范：可读性强，便于管理与排查问题

## 功能使用

### 功能说明

在UGC开发中，常见的实例引用方式包括：

- 手动维护对象引用
- 编写复杂查找逻辑（如遍历、标签匹配等）
- 场景变更后需同步修改代码

Actor实例路径机制提供了更高效的替代方案：

- 编辑器中右键选中实例后一键复制路径
- 运行时通过路径直接定位 Actor
- 无需维护额外引用关系

### 路径格式

实例路径采用统一格式：
`<LevelName>.<ActorInstanceName>`

示例：
`"UGCmap.BP_SpawnerElite4"`

字段说明：
|组成部分|说明|示例|
|:-:|-|-|
|`LevelName`|关卡 / 世界名称|`UGCmap`|
|`ActorInstanceName`|实例在关卡中的对象名|`BP_SpawnerElite4`|

### 编辑器使用方式

操作步骤：
1.在编辑器场景视图中中选中目标 Actor
2.右键打开对象的上下文菜单

### 运行时使用方式

在运行时通过`UGCActorComponentUtility.GetActorByActorInstancePath`来获取实例。
```lua
-- 通过实例路径获取 Actor
local Actor = UGCActorComponentUtility.GetActorByActorInstancePath("UGCmap.test_8")

-- 验证 Actor 有效性并使用
if Actor and UGCObjectUtility.IsObjectValid(Actor) then
    ugcprint_concat("Find Actor Success", Actor:GetName())
    -- 后续可以对该 Actor 进行操作
end
```

### 使用注意事项

- 有效性校验：使用`UGCObjectUtility.IsObjectValid()`
- 多子关卡支持：若地图中有多个子关卡，接口会遍历所有关卡
- 对象生命周期：确保 Actor 已完成创建与初始化

## 典型应用场景

场景一：Lua 配置引用固定实例
```lua
-- 在 Lua 配置中直接引用场景中的目标对象
local TargetActorPath = "UGCmap.Dynamic_ShootingRange_Target5"
local TargetActor = UGCActorComponentUtility.GetActorByActorInstancePath(TargetActorPath)

if TargetActor then
    -- 对目标对象进行配置或操作
    TargetActor:SetActorLocation(Vector.New(100, 100, 100))
end
```

场景二：查找关卡检查点
```lua
-- 关卡脚本通过路径查找预设的点位
local CheckpointPath = "UGCmap.CheckPoint_1"
local CheckpointActor = UGCActorComponentUtility.GetActorByActorInstancePath(CheckpointPath)

if CheckpointActor then
    -- 检查玩家是否到达检查点
    local Player = UGCPlayerController.GetPawn()
    if Player then
        local Distance = Player:GetDistanceTo(CheckpointActor)
        if Distance < 10.0 then
            -- 到达检查点，触发相关逻辑
        end
    end
end
```

场景三：触发器/机关控制
```lua
-- 通过路径查找场景中的触发器或机关
local TriggerPath = "UGCmap.Trigger_Gate1"
local TriggerActor = UGCActorComponentUtility.GetActorByActorInstancePath(TriggerPath)

if TriggerActor then
    -- 监听触发器事件或进行激活操作
    TriggerActor:Activate()
end
```

场景四：批量处理实例
```lua
-- 批量复制多个实例路径后，在脚本中统一处理
local TargetPaths = {
    "UGCmap.Target1",
    "UGCmap.Target2",
    "UGCmap.Target3"
}

for i, Path in ipairs(TargetPaths) do
    local TargetActor = UGCActorComponentUtility.GetActorByActorInstancePath(Path)
    if TargetActor then
        -- 对每个目标进行统一操作
        TargetActor:SetActorScale3D(Vector.New(2, 2, 2))
    end
end
```
