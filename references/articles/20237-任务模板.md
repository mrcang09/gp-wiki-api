# 任务模板

任务模板是为游戏开发者提供的一套完整的任务系统解决方案，包含界面展示、任务逻辑、奖励发放等核心功能，模板预置了多种常见任务类型（如击杀、收集、在线时长），并支持成长型与活跃型两种主流任务线模式，能帮助开发者快速构建规范的任务体系，显著提升开发效率。

<br>

## 功能概要

任务模板功能包含任务读取、任务进度更新、奖励领取等部分，这些功能的执行流程分别如下：
- 任务读取时，系统读取绑定的任务线表，并依据其中存储的任务ID生成对应的任务实例，最终在任务界面中向玩家展示
- 任务进度更新中，当玩家进行游戏时，系统会实时更新玩家数据，并自动同步更新所有关联任务的进度状态，无需玩家手动操作
- 奖励领取功能包括任务奖励领取和活跃度奖励领取共两种，所有领取的奖励均以物品形式发放至玩家账户

## 配置依赖

任务模板的运行依赖于 ``GamePart`` 的特定配置、 ``TaskTemplateComponent`` 组件和 ``任务线表`` ，需要前置将其正确配置好。

### GamePart

在编辑器主界面的工具栏，点击进入【玩法通用设置】。

![image.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/PtSOoimage.png)

点击【Active Game Part Configs】右侧的加号，添加并选择需要启用的GamePart模块。

![任务模板-添加模块.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/umha4%E4%BB%BB%E5%8A%A1%E6%A8%A1%E6%9D%BF-%E6%B7%BB%E5%8A%A0%E6%A8%A1%E5%9D%97.drawio.png)

任务模板需要任务以及任务线，因此需要启用 ``GP_TaskManager`` 模块，在该模块配置任务和任务线。

![ScreenShot_2025-10-11_104951_010.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/KAm4NScreenShot_2025-10-11_104951_010.png)

完成任务后会获得奖励或活跃值，它们的本质为虚拟物品，需要启用 ``GP_VirtualItemManager`` 模块。

![ScreenShot_2025-10-11_103319_258.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/QOqxXScreenShot_2025-10-11_103319_258.png)

### 任务线表

任务模板会读取任务线表，显示对应名称的任务线。默认不存在任务线表，需要创建结构体为 ``FTaskTemplateConfig`` 的表格，并绑定到 ``TaskTemplateComponent`` 组件。

> TaskTemplateComponent组件位置： ``ExtendResource/TaskTemplate/OfficialPackage/Asset/Task/Blueprint/TaskTemplateComponent``

![ScreenShot_2025-10-14_104307_481.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/basdTScreenShot_2025-10-14_104307_481.png)

|属性名称|说明|
|:-:|-|
|任务ID|任务的唯一标识符|
|任务名称|任务的核心标识与主要标题|
|任务类型|定义任务的分类（如攻击型、收集型等），影响任务的完成逻辑，详细配置见 [任务类型配置](https://developer.gp.qq.com/wikieditor/#/catalog/20237?autoJump=%E4%BB%BB%E5%8A%A1%E7%B1%BB%E5%9E%8B%E9%85%8D%E7%BD%AE)|
|任务说明|对任务目标或完成条件的描述文本|
|任务奖励列表|用于配置任务完成后可获得的奖励物品列表，一个数组元素对应一种奖励物品，并包含该物品的【物品ID】和【物品数量】|
|物品ID|在【任务奖励列表】中指定奖励物品的ID，需填入表格 ``UGCObject`` 中对应的的【项目ItemID】|
|物品数量|在【任务奖励列表】中指定对应奖励物品的数量|
|开始时间|任务生效的起始时间，在此时间之前，任务不可见且无法完成|
|结束时间|任务生效的截止时间，在此时间之后，任务过期且无法完成|
|过期后是否显示|控制任务过期时是否仍在界面中显示|
|是否显示前往按钮|控制在该任务未完成时，是否在界面显示“前往”按钮|

### 任务类型配置

配置任务期间需要配置任务的类型，任务类型有多种，可根据个人需求，先选择任务类型，再选择具体任务，最后根据具体任务配置相关属性。

![任务模板-任务类型配置.drawio.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/N8qQB%E4%BB%BB%E5%8A%A1%E6%A8%A1%E6%9D%BF-%E4%BB%BB%E5%8A%A1%E7%B1%BB%E5%9E%8B%E9%85%8D%E7%BD%AE.drawio.png)

任务类型说明如下：

|任务类型|说明|
|:-:|-|
|UGCAttackTaskType|攻击型任务，包含累计击杀(BP_UGCCumulativeKillNumTask)和累计造成枪械伤害(BP_UGCCumulativeWeaponDemageTask)|
|UGCEvaluationTaskType|条件型任务，包含获得特定名次(BP_UGCAchieveRankingTask)|
|UGCltemTaskType|物品型任务，包含累计消耗虚拟物品(BP_UGCCumulativeCostItemTask)和累计获得虚拟物品(BP_UGCCumulativeGetItemTask)|
|UGCSystemTaskType|系统型任务，包含累计在线(BP_UGCCumulativeOnlineTimeTask)和达成任务线进度(BP_UGCTaskLineProgressTask)|

|活跃任务线配置|定义活跃任务线中所包含的任务，一个数组元素对应一个任务，并包含该任务的【任务ID】和【任务排序优先级】|
|任务ID|在【活跃任务线配置】中对应的任务ID，需填入任务配置中对应的【任务ID】|
|任务排序优先级|在【活跃任务线配置】中设定，用于控制对应任务在界面列表中的显示位置，数值越小，位置越靠前。|
|进度奖励列表|定义活跃度达到特定数值时可领取的进度奖励，一个数组元素对应一个进度奖励，并包含该进度奖励的【活跃度】和【奖励道具列表】|
|活跃度|在【进度奖励列表】中设定，表示领取该进度奖励所需达到的活跃度|
|奖励道具列表|在【进度奖励列表】中设定，定义达到对应活跃度后可获得的具体奖励道具，一个数组元素对应一种奖励道具，并包含该奖励道具的【物品ID】和【物品数量】|
|物品ID|在【奖励道具列表】中指定奖励道具的ID，需填入表格 ``UGCObject`` 中对应的的【项目ItemID】|
|物品数量|在【奖励道具列表】中指定对应奖励道具的数量|
|活跃任务线重置类型|设定任务线的重置方式，可选择“不重置”、“每日重置”和“每周重置”|
|活跃任务线重置时间|指定活跃任务线每日重置的具体时间，【活跃任务线重置类型】选择“每日重置”时生效|
|活跃任务线周重置类型|指定活跃任务线在每周的重置日期，可选择“周一”至“周日”，【活跃任务线重置类型】选择“每周重置”时生效|
|活跃度道具ID|用于计算活跃度数值的道具ID，该道具的累积数量即为活跃度，需填入表格 ``UGCObject`` 中对应的的【项目ItemID】|
|开始时间|整个任务线生效的起始时间，在此时间之前，任务线不可见且无法完成|
|结束时间|整个任务线生效的截止时间，在此时间之后，任务线不可见且无法完成|

## 功能调试

模板内置了一个简易的调试控件，方便开发者进行功能调试。

> 控件蓝图路径：ExtendResource/TaskTemplate/OfficialPackage/Asset/Task/Blueprint/WBP_TaskMainUIButton

![ScreenShot_2025-10-16_162048_904.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/5alahScreenShot_2025-10-16_162048_904.png)

将控件通过 [AddToViewport](https://developer.gp.qq.com/api/#/searchContent/UUserWidget?classDetailShow=true&path=class%2Fdetail%2FOthers%2FUUserWidget.json&isSelect=1&apiEnc=%5B%22%E7%B1%BB%22%5D&apiLabel=UUserWidget&autoJump=AddToViewport) 添加到屏幕视口中，启动PIE调试后出现测试UI界面。

```lua

--UGCGameState.lua部分代码
function UGCGameState:ReceiveBeginPlay()
    self.SuperClass.ReceiveBeginPlay(self);
    if self:HasAuthority() == true then 
        -- 只有客户端加载UI
    else
        local TaskBtnClass = UGCObjectUtility.LoadClass(UGCGameSystem.GetUGCResourcesFullPath("ExtendResource/TaskTemplate/OfficialPackage/Asset/Task/Blueprint/WBP_TaskMainUIButton.WBP_TaskMainUIButton_C"));
        local TaskBtn = UGCWidgetManagerSystem.CreateWidget(TaskBtnClass);
        TaskBtn:AddToViewport();
    end
end
```

![ScreenShot_2025-10-16_165129_146.png](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/242vZScreenShot_2025-10-16_165129_146.png)

**进入任务界面**

点击【打开任务主界面】按钮后，系统将默认打开首个任务线的界面。可通过右侧的“任务线列表”随时切换查看其他任务线。

![打开主界面.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/B0tus%E6%89%93%E5%BC%80%E4%B8%BB%E7%95%8C%E9%9D%A2.gif)

**重置活跃任务线**

在 ``TaskLineName`` 中输入活跃型任务线的名称，点击【重置活跃任务线】按钮后，该任务线的活跃度与所有任务状态将会被重置。

![重置活跃任务线.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/ZkpqL%E9%87%8D%E7%BD%AE%E6%B4%BB%E8%B7%83%E4%BB%BB%E5%8A%A1%E7%BA%BF.gif)

**更新任务线进度**

在 ``TaskLineName`` 中输入任务线的名称，在 ``TaskLineProgress`` 输入数值，点击【更新任务线进度】按钮后，如果这个任务线为成长型，该任务线的等级将提升至 ``TaskLineProgress`` 的对应数值，并解锁小于等于该等级的阶段；如果这个任务线为活跃型，该任务线的活跃值将提升至 ``TaskLineProgress`` 的对应数值。

![更新成长任务线.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/f5IUm%E6%9B%B4%E6%96%B0%E6%88%90%E9%95%BF%E4%BB%BB%E5%8A%A1%E7%BA%BF.gif)
![更新活跃任务线.gif](https://cgugc-video-test-1258633575.cos.ap-shanghai.myqcloud.com/wiki_picture/vzaQ5%E6%9B%B4%E6%96%B0%E6%B4%BB%E8%B7%83%E4%BB%BB%E5%8A%A1%E7%BA%BF.gif)

**更新成长型任务进度**

在 ``TaskLineName`` 中输入成长型任务线的名称，然后在 ``LevelIndex`` 输入阶段的位置（输入2表示第二个阶段）， ``TaskIndex`` 输入任务的位置（输入2表示第二个任务）， ``TaskProgress`` 输入设置的任务进度，最后点击【更新任务进度】，即可改变成长型任务中对应任务的进度。
