# 玩家控制器与交互详解
##  系统职责

UGCPlayerController 是玩家与游戏交互的核心控制器，负责：

* ✅ 队伍切换请求处理
* ✅ 装备配置修改（武器/配件）
* ✅ UI 加载与初始化（客户端）
* ✅ RPC 服务端函数定义
* ✅ 武器列表组织与管理
* ✅ 输入处理与玩家控制

##  核心数据结构

```
local UGCPlayerController = {
    LastWeaponListIndex = 10   -- 上次编辑的武器索引
}
```

**说明**：

* LastWeaponListIndex：记录玩家最后编辑的武器槽位，用于清理无效槽位

##  生命周期函数

### ReceiveBeginPlay()

**调用时机**：玩家控制器初始化时

**功能**：

* 服务器端：初始化武器配置
* 客户端：加载主 UI 布局和小地图

```
function UGCPlayerController:ReceiveBeginPlay()
    UGCPlayerController.SuperClass.ReceiveBeginPlay(self)
    
    if self:HasAuthority() then
        -- 服务器端：初始化默认武器配置
        self:InitWeaponEquipConfig()
    else

##  队伍管理

### ServerChangeTeam(NewTeamID, bForce)

**RPC 函数**：客户端调用，服务器执行

**参数**：

* NewTeamID：目标队伍 ID（Red=1, Blue=2）
* bForce：是否强制切换（忽略阶段限制）

**功能**：

1. 获取游戏模式
2. 检查队伍 ID 有效性
3. 检查是否在准备阶段或强制切换
4. 调用游戏模式的队伍切换逻辑

**调用条件**：

* 目标队伍为红队或蓝队
* 当前在准备阶段 或 bForce 为 true

```
function UGCPlayerController:ServerChangeTeam(NewTeamID, bForce)
    local GameMode = UGCGameSystem.GetGameMode()
    
    ugcprint("UGCPlayerController:ServerChangeTeam "..tostring(self.PlayerName)
        .." TeamID = "..tostring(self.TeamID).." NewTeamID = "..tostring(NewTeamID) 
        .. "GameMode.GameState.CurrentGameStatus="..tostring(GameMode.GameState.CurrentGameStatus))
    
    if GameMode then
        -- 检查是否可以切换队伍
        if NewTeamID ~= self.TeamID and 
           (GameMode.GameState.CurrentGameStatus == EGameStatus.Prepare or bForce) then
            ugcprint("UGCPlayerController:ServerChangeTeam ChangeTeamFinish.......1111")
            GameMode:PlayerChangeTeam(self.PlayerKey, NewTeamID)
        end
    else
        ugcprint("UGCPlayerController:ServerChangeTeam ChangeTeamFinish")
    end
end
```

### ServerChangeWeapons(WeaponTable)

**RPC 函数**：客户端调用，服务器执行

**参数**：

* WeaponTable：完整的武器配置表

**功能**：

1. 验证参数类型
2. 检查是否在准备阶段
3. 设置新的武器列表

**调用条件**：

* WeaponTable 必须为 table 类型
* 当前游戏阶段为准备阶段

**数据结构示例**：

```
WeaponTable = {
    [1] = {
        ItemID = 1001,           -- 武器 ID
        Count = 1,              -- 武器数量
        SubItemList = {
            [1] = {ItemID = 2001, Count = 1},  -- 配件 1
            [2] = {ItemID = 2002, Count = 1}   -- 配件 2
        }
    },
    [2] = {
        ItemID = 1002,
        Count = 1,
        SubItemList = {
            [1] = {ItemID = 2001, Count = 1}
        }
    }
}
```

```
function UGCPlayerController:ServerChangeWeapons(WeaponTable)

### ServerChangeWeaponList(Index, SubIndex, NewID, NewCount)

**RPC 函数**：客户端调用，服务器执行

**参数**：

* Index：武器槽位索引（≥1）
* SubIndex：配件槽位索引（0=主武器，≥1=配件）
* NewID：新的物品 ID（0=清空）
* NewCount：新的物品数量（可选）

**功能**：

1. 检查是否在准备阶段
2. 记录最后编辑的武器槽位
3. 根据索引类型更新主武器或配件
4. 如果索引超出当前列表长度，扩展列表
5. 同步武器列表到玩家状态

**调用条件**：

* 当前游戏阶段为准备阶段
* Index 为有效正整数

**参数说明**：

|参数|说明|示例|
|-----|-----|-----|
Index|武器槽位（1=第1把武器，2=第2把武器...）|1|
SubIndex|配件槽位（0=主武器，1=配件1，2=配件2...）|0|
NewID|新的物品 ID（0=清空，-1=占位）|1001|
NewCount|新的物品数量（可选）|1|

**逻辑图解**：

```
WeaponList[1] = {
    ItemID = 1001,        ← SubIndex=0 修改此处
    Count = 1,
    SubItemList = {
        [1] = {ItemID = 2001, Count = 1},  ← SubIndex=1 修改此处
        [2] = {ItemID = 2002, Count = 1}   ← SubIndex=2 修改此处
    }
}
```

```
function UGCPlayerController:ServerChangeWeaponList(Index, SubIndex, NewID, NewCount)
    local GameMode = UGCGameSystem.GetGameMode()
    
    if GameMode and GameMode.GameState.CurrentGameStatus == EGameStatus.Prepare then
        local WeaponList = self.PlayerState.WeaponList
        local WeaponListNum = #WeaponList
        
        self.LastWeaponListIndex = Index
        
        -- 默认 ItemID 为 0（清空）
        if NewID == nil then
            NewID = 0
        end
        
        -- 修改现有武器
        if Index > 0 and Index <= WeaponListNum then
            ugcprint("WeaponListNum "..tostring(WeaponListNum))
            local WeaponConfig = WeaponList[Index]
            local SubItemListNum = #WeaponConfig.SubItemList
            
            -- 修改配件
            if SubIndex > 0 then
                ugcprint("SubItemListNum "..tostring(SubItemListNum))
                
                if SubIndex <= SubItemListNum then
                    -- 修改现有配件
                    ugcprint("SubItemListNum SubIndex <= SubItemListNum")
                    local SubItem = WeaponConfig.SubItemList[SubIndex]
                    SubItem.ItemID = NewID
                    if NewCount and NewCount > 0 then
                        SubItem.Count = NewCount
                    end
                else
                    -- 扩展配件列表
                    ugcprint("SubItemListNum SubIndex > SubItemListNum")
                    if NewCount and NewCount > 0 then
                        for i = SubItemListNum + 1, SubIndex do
                            if i == SubIndex then
                                -- 添加新配件
                                table.insert(WeaponConfig.SubItemList, {ItemID = NewID, Count = NewCount})
                            else
                                -- 添加占位符
                                table.insert(WeaponConfig.SubItemList, {ItemID = -1, Count = -1})
                            end
                        end
                    end
                end
            else
                -- 修改主武器
                WeaponConfig.ItemID = NewID
                if NewCount and NewCount > 0 then
                    WeaponConfig.Count = NewCount
                end
            end
            
            self.PlayerState:SetWeaponList(WeaponList)
            
        elseif Index > WeaponListNum then
            -- 添加新武器槽位
            ugcprint("WeaponListNum Index > WeaponListNum"..tostring(Index))
            WeaponList[Index] = {}
            WeaponList[Index].ItemID = NewID
            if NewCount and NewCount > 0 then
                WeaponList[Index].Count = NewCount
            end
            if 0 == SubIndex then
                WeaponList[Index].SubItemList = {}
            end
            
            self.PlayerState:SetWeaponList(WeaponList)
        end
    end
end
```

### OrganizeWeaponList()

**功能**：清理无效的武器槽位

**处理逻辑**：

1. 获取当前武器列表
2. 清理 LastWeaponListIndex 之后的所有槽位
3. 移除玩家编辑过程中产生的无效数据

**使用场景**：

* 玩家复活时，清理上次编辑留下的无效槽位
* 确保武器列表整洁，避免发放无效装备

```
function UGCPlayerController:OrganizeWeaponList()
    local WeaponList = self.PlayerState.WeaponList
    
    for i = self.LastWeaponListIndex + 1, #WeaponList do
        WeaponList[i] = nil
    end
end
```

**原代码路径**：Script/Blueprint/UGCPlayerController.lua:138-144

**使用方式演示：**

```

##  开发建议

### 1. 添加新的 RPC 函数

如需添加新的客户端→服务器通信函数：

1. 在 GetAvailableServerRPCs() 中注册函数名
2. 实现函数逻辑（以 Server\_ 开头）

**添加快速聊天功能**

```
-- 1. 注册 RPC
function UGCPlayerController:GetAvailableServerRPCs()
    return "ServerChangeTeam", "ServerChangeWeaponList", "ServerChangeWeapons", "ServerSendQuickChat"
end

-- 2. 实现函数
function UGCPlayerController:ServerSendQuickChat(ChatID)
    local GameMode = UGCGameSystem.GetGameMode()
    if GameMode then
        -- 广播聊天消息给所有玩家
        for _, PlayerKey in ipairs(GameMode.PlayerKeyList) do
            local PC = UGCGameSystem.GetPlayerControllerByPlayerKey(PlayerKey)
            if PC then
                -- 通知客户端显示聊天消息
                UGCEventSystem:SendEventToClient(PC, CustomEvent.QuickChatReceived, {
                    PlayerName = self.PlayerName,
                    ChatID = ChatID
                })
            end
        end
    end
end

-- 3. 客户端调用
function UI_QuickChatButton_OnClick(ChatID)
